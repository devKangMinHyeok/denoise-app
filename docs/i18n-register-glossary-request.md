# 요청: Vocast 한국어 레지스터·용어 확정 (디자인/DS 유지보수 에이전트용)

> 이 문서는 랜딩 한국어 배선을 마친 상태에서 **디자인 시스템 쪽에 넘기는 결정 요청서**다.
> 결정이 나오면 랜딩 카피(필요 시)를 재작업해 다시 배선한다. 급한 항목은 아니며,
> 사소한 카피 표현 차이는 지금 상태로 두어도 무방하다.

## 배경 (현재 배선 상태)

- 랜딩(마케팅 사이트)에 한국어 배선 완료. 소스: `vocast-landing-i18n.json` 핸드오프 확정본.
- 현재 랜딩 카피 레지스터: **합니다체**(정중한 서술) + 헤드라인/네비/버튼/배지는 간결한 명사형·구어체.
- denoise 표기: 랜딩 = **노이즈 제거**.
- 맥 앱 UI는 별도 한국어화돼 있고, Timbre DS 가이드는 **해요체 전면 적용**을 규정한다.

## 결정해야 할 것

### 1. 표면별 레지스터 정책 (가장 중요)

충돌: **앱 UI = 해요체**(DS 가이드) vs **마케팅 랜딩 = 합니다체**(로컬라이제이션 지시).

셋 중 하나로 확정하고 DS 가이드에 "표면 → 레지스터" 표로 못박아 달라:

- **A안** 앱 = 해요체 / 마케팅 = 합니다체 (표면 분리). ← 핸드오프 권장안, 현재 배선과 일치
- **B안** 전 표면 해요체 통일 → 랜딩 카피 재작성 필요
- **C안** 전 표면 합니다체 통일 → 앱 카피 재작성 필요

산출물: DS 문서에 표면별 레지스터 매트릭스 + 근거 한 줄. B/C안이면 해당 표면 카피를 재작업해 전달.

### 2. 이중언어 용어집(글로서리) 확정

현재 랜딩에서 쓰는 표기(검증·확정 요청):
보이스 클로닝 · 온디바이스 · 낭독 · 노이즈 제거 · 보이스프린트 · 운율 · 원고 · 크리에이터 · 한 번 결제.

- **알려진 불일치**: denoise → 마케팅 "노이즈 제거" vs 앱 chrome "잡음 제거".
  하나로 확정하거나 표면별 허용을 명시.
- 영어 유지(두 언어 공통, 번역 금지): Vocast, MCP, Model Context Protocol, macOS,
  Apple Silicon, M1, Metal, stdio, CI, PNS, 툴 이름(denoise/clone_voice/…), MyVoice, ETA, $49.

산출물: `영어 → 확정 한국어(+표면별 예외)` 용어표.

### 3. 재배선이 필요할 때의 산출물 형식

레지스터/용어가 바뀌어 랜딩 카피를 고쳐야 하면 **같은 스키마의 `vocast-landing-i18n.json`을
업데이트해서** 넘겨라. 코딩 쪽은 그 JSON을 그대로 재배선한다.

- 키 이름·중첩·배열 순서 고정. `{price}` 토큰 유지(코드가 실제 가격으로 치환).
- 느낌표·이모지·긴 대시(U+2014/U+2015) 금지. 구분자는 가운뎃점 `·`.

## 바꾸지 말 것 (확정)

- URL 구조(영어 루트 `/`, 한국어 `/ko/`), hreflang/canonical/og:locale, 라우팅.
- 네비 항목 구성: 기능 · 품질 · 프라이버시 · AI (MCP) · 도구 · 가격 · 블로그.
